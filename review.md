# Review — критерии проверки Agent Office

> Для reviewer-профиля. Проверяй по пунктам. **Каждую находку — верифицируй**
> (открой файл/запусти), не доверяй догадке: лучше пропустить, чем зарепортить
> ложное срабатывание и заставить билдера «чинить» то, что не сломано.

## Сборка и упаковка

- [ ] Проект собирается без ошибок в `dashboard/dist/index.js` (IIFE).
- [ ] **React НЕ забандлен** в плагин (берётся из `window.__HERMES_PLUGIN_SDK__`);
      бандл компактный.
- [ ] `manifest.json` валиден и соответствует актуальной схеме (сверь с
      example-dashboard): есть `name`, `label`, `entry`, конфигурация вкладки.

## Работа в дашборде

- [ ] Вкладка «Agent Office» появляется, кликается, рендерится.
- [ ] В консоли браузера нет ошибок; `window.__HERMES_PLUGINS__` определён.
- [ ] В Network нет 404 на `manifest.json` / `index.js` / css.

## Функционал (MVP)

- [ ] Профили подтягиваются из **реального** API (не хардкод/мок).
- [ ] Статус агента (`idle`/`working`/`blocked`) отражает реальные данные kanban/status.
- [ ] Клик по персонажу показывает **реальную** активность агента (задача, runs/логи).
- [ ] Есть состояния loading / error / empty.

## Гигиена

- [ ] Нет закоммиченных секретов/ключей (`.env`, токены).
- [ ] Нет мёртвого кода и закомментированных кусков «на всякий».
- [ ] README/spec не разошлись с тем, что реально сделано.

## Вывод ревью

Если есть несоответствие между задумкой (spec) и реализацией — переведи задачу в
**Blocked** через `kanban_block` и опиши, что именно согласовать. Иначе — `kanban_complete`
с кратким саммари: что проверено, что ок, что улучшить.
